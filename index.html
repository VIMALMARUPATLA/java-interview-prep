Core Java (20 Questions)

1. What are the main features of Java?
Java is a high-level, object-oriented programming language with features like: Platform Independence (WORA via JVM), Object-Oriented (encapsulation, inheritance, polymorphism, abstraction), Robust (memory management, exception handling), Secure (no pointers, sandboxed execution), Multithreaded, Simple (no pointers), Portable, and High Performance (JIT compiler).
2. Explain the difference between JDK, JRE, and JVM.
JDK (Java Development Kit): Software development kit with JRE, compiler (javac), and tools. JRE (Java Runtime Environment): Runtime environment with JVM and libraries to run apps. JVM (Java Virtual Machine): Interprets bytecode, manages memory, ensures platform independence.
3. What is the purpose of the final keyword in Java?
The final keyword: For variables, makes them constants; for methods, prevents overriding; for classes, prevents inheritance (e.g., String).
4. What are wrapper classes in Java?
Wrapper classes (e.g., Integer, Double) provide object representations of primitives (e.g., int, double). Used in collections and utility methods (e.g., Integer.parseInt()).
5. Explain the difference between == and .equals().
==: Compares references for objects or values for primitives. .equals(): Compares content (e.g., String values), overridden in classes like String.
6. What is autoboxing and unboxing in Java?
Autoboxing: Converts primitives to wrapper objects (e.g., int to Integer). Unboxing: Converts wrapper objects to primitives (e.g., Integer to int). Example: Integer i = 10; int j = i;
7. Explain method overloading and method overriding.
Overloading: Same method name, different parameters (compile-time). Overriding: Subclass redefines superclass method (runtime).
8. What are access modifiers in Java?
Control visibility: public (everywhere), protected (package + subclasses), default (package), private (class only).
9. What is the difference between String, StringBuilder, and StringBuffer?
String: Immutable, thread-safe. StringBuilder: Mutable, not thread-safe, fast. StringBuffer: Mutable, thread-safe, slower.
10. What is the purpose of the static keyword?
Static: For variables, class-level shared; for methods, callable without instance; for blocks, runs at class loading.
11. Explain the difference between this and super keywords.
this: Refers to current instance, distinguishes fields. super: Refers to superclass, calls methods/constructors.
12. What is the difference between an interface and an abstract class?
Interface: Fully abstract, multiple inheritance, constants only. Abstract Class: Partial implementation, single inheritance, instance variables allowed.
13. What is a constructor, and what are its types?
Constructor initializes objects. Types: Default (no params), Parameterized (with params), Copy (copies another).
14. How does garbage collection work in Java?
JVM’s GC reclaims memory from unreferenced objects using Mark-and-Sweep or Generational Collection (Young/Old).
15. What is the purpose of try-catch-finally in exception handling?
try: Code that might throw exceptions. catch: Handles exceptions. finally: Executes regardless (cleanup).
16. Explain checked and unchecked exceptions.
Checked: Must be handled/declared (e.g., IOException), compile-time. Unchecked: RuntimeException subclasses (e.g., NullPointerException).
17. What are functional interfaces in Java?
Single abstract method interfaces (e.g., Runnable), used with lambdas, annotated @FunctionalInterface.
18. What is the volatile keyword in Java?
Ensures variable visibility across threads, prevents caching in multithreading.
19. What is the difference between break, continue, and return?
break: Exits loop/switch. continue: Skips iteration. return: Exits method.
20. What are Java annotations and how are they used?
Metadata (e.g., @Override) for compiler, runtime, or frameworks (e.g., Spring).

OOP (10 Questions)

1. What are the four pillars of Object-Oriented Programming?
Encapsulation, Inheritance, Polymorphism, Abstraction.
2. What is encapsulation in Java?
Bundling data and methods, restricting access via private fields and public getters/setters.
3. What is inheritance and its types?
Reusing code via parent-child. Types: Single, Multilevel, Hierarchical (Java doesn’t support multiple class inheritance).
4. Explain polymorphism with examples.
Same interface, different behavior. Compile-time: Overloading (e.g., add(int, int) vs add(double, double)). Runtime: Overriding (e.g., Animal.speak()).
5. What is an abstract class, and when should it be used?
Can’t be instantiated, may have abstract/concrete methods. Use for shared partial implementation (e.g., Animal).
6. What is an interface, and how does it differ from an abstract class?
Defines contract, supports multiple inheritance. Unlike abstract classes, no instance variables or concrete methods (pre-Java 8).
7. What is method overriding?
Subclass redefines superclass method with same signature, using @Override.
8. Explain the concept of aggregation and composition.
Aggregation: Weak “has-a” (e.g., Car has Wheels). Composition: Strong ownership (e.g., House has Rooms).
9. How does Java support multiple inheritance?
Via interfaces (class can implement multiple), not classes, to avoid diamond problem.
10. What is dependency injection in OOP?
Providing dependencies to a class (e.g., via constructor), improves modularity (e.g., Spring @Autowired).

Java 8 (40 Questions)

1. What is a functional interface, and why do we need it?
A functional interface is an interface with exactly one abstract method (SAM - Single Abstract Method). It’s used as a target type for lambda expressions and method references in Java 8. We need it to enable functional programming by allowing methods to accept behavior (via lambda) rather than just data.
2. What is the difference between Consumer and Supplier interfaces?
Consumer: Accepts a single input (T) and performs an operation without returning a result (e.g., void accept(T t)). Supplier: Takes no input and produces a result of type T (e.g., T get()). In short, Consumer consumes data, Supplier supplies it.
3. What is CompletableFuture in Java 8?
CompletableFuture is a class that enhances Java’s Future by supporting asynchronous programming with a fluent API. It allows chaining operations (e.g., thenApply, thenAccept) and handling results or exceptions once a computation completes, without blocking.
4. Why are default and static methods introduced in Java 8?
Default methods: Allow adding new methods to interfaces without breaking existing implementations (backward compatibility). They provide a default implementation. Static methods: Enable utility methods in interfaces, reducing the need for companion utility classes (e.g., Comparator.comparing()).
5. When we have interfaces and abstract classes, why do we need functional interfaces?
Functional interfaces are specifically designed for lambda expressions and functional programming. Unlike regular interfaces or abstract classes, they enforce a single responsibility (one abstract method), making them ideal for passing behavior as data.
6. Write a custom functional interface and call it using lambda expressions. 

        @FunctionalInterface
        interface Calculator {
            int calculate(int a, int b);
        }
        public class Main {
            public static void main(String[] args) {
                Calculator add = (a, b) -> a + b;
                System.out.println(add.calculate(5, 3)); // Output: 8
            }
        }
7. What is a lambda expression?
A lambda expression is a concise way to represent an anonymous function (behavior) that can be passed around. It consists of parameters, an arrow (->), and a body (e.g., (x, y) -> x + y). It’s used with functional interfaces.
8. Can we use lambda expressions with normal interfaces?
No, lambda expressions can only be used with functional interfaces (those with one abstract method). Normal interfaces with multiple abstract methods don’t provide a clear target for lambda implementation.
9. What is the try-with-resources feature?
Introduced in Java 7 and enhanced in Java 8, try-with-resources automatically closes resources (e.g., files, sockets) that implement AutoCloseable or Closeable. It simplifies resource management and reduces boilerplate code. Example: 

        try (FileReader fr = new FileReader("file.txt")) {
            // Use fr
        } // Automatically closed
10. Have you worked with Streams? Why do we need them?
Yes, Streams provide a functional way to process collections of data declaratively. We need them for concise, readable code, lazy evaluation, and easy parallelization (e.g., parallelStream()).
11. What are intermediate functions in Streams, and what do they return?
Intermediate operations (e.g., filter(), map(), sorted()) transform or filter a Stream and return another Stream. They are lazy and don’t execute until a terminal operation is called.
12. What are terminal functions in Streams, and what do they return?
Terminal operations (e.g., collect(), forEach(), reduce()) trigger the Stream processing and produce a result (e.g., a collection, a single value) or a side effect. They return non-Stream types or void.
13. Explain Java 8 enhancements in collections.
Java 8 added methods like forEach(), stream(), parallelStream(), and removeIf() to the Collection interface. It also introduced default methods in Iterator and List (e.g., replaceAll()), enhancing functionality.
14. What is Optional in Java 8?
Optional is a container object that may or may not contain a non-null value. It helps avoid NullPointerException by forcing explicit null handling (e.g., isPresent(), orElse()).
15. Explain Stream API and its benefits.
The Stream API enables processing sequences of elements (e.g., collections) in a functional style. Benefits include: concise code, lazy evaluation, easy parallel processing, and pipeline operations (filter, map, etc.).
16. How does the new Date and Time API work in Java 8?
The java.time package replaces older APIs (Date, Calendar) with immutable, thread-safe classes like LocalDate, LocalTime, LocalDateTime, and ZonedDateTime. It uses a fluent API and supports time zones, durations, and periods.
17. How do lambda expressions improve code readability?
Lambda expressions reduce boilerplate code (e.g., anonymous classes) and focus on the core logic, making code shorter, clearer, and more expressive (e.g., (x) -> x * 2 vs. a full class).
18. Explain how method references work in Java 8.
Method references (::) are a shorthand for lambda expressions that call an existing method. Types include: Static (Class::staticMethod, e.g., Math::max), Instance (object::instanceMethod), Constructor (Class::new). Example: list.forEach(System.out::println).
19. How does Java 8 handle parallel processing with Streams?
Java 8 provides parallelStream(), which splits a Stream into multiple threads using the Fork/Join framework. It simplifies parallelism but requires careful use to avoid overhead or thread-safety issues.
20. What are the performance benefits of using Streams?
Streams offer lazy evaluation (processing only when needed), efficient pipelining (fusing operations), and parallel execution with parallelStream(), improving performance for large datasets.
21. Explain the internal working of HashMap in Java 8.
In Java 8, HashMap uses a bucket array with linked lists or red-black trees (if a bucket exceeds 8 nodes). This improves performance for high-collision cases from O(n) to O(log n) for operations like get() and put().
22. How does Java 8 improve concurrency with CompletableFuture?
CompletableFuture supports non-blocking, asynchronous tasks with callbacks (e.g., thenApply(), thenCombine()). It simplifies chaining operations and exception handling compared to Future.
23. How does Java 8 handle functional-style programming?
Java 8 introduces lambdas, functional interfaces, Streams, and method references, enabling declarative, immutable, and behavior-passing paradigms typical of functional programming.
24. What are the different functional interfaces in Java 8?
Key ones in java.util.function: Function (Takes T, returns R), Consumer (Takes T, returns void), Supplier (Takes nothing, returns T), Predicate (Takes T, returns boolean). Others: BiFunction, UnaryOperator, etc.
25. How do you convert a list to a map using Streams? 

        List list = Arrays.asList("a", "b", "c");
        Map map = list.stream()
            .collect(Collectors.toMap(s -> s, String::length));
        // Output: {a=1, b=1, c=1}
26. How do you filter and sort data using Streams? 

        List numbers = Arrays.asList(5, 2, 9, 1);
        List result = numbers.stream()
            .filter(n -> n > 2)
            .sorted()
            .collect(Collectors.toList());
        // Output: [5, 9]
27. How does Java 8 handle multiple inheritance with default methods in interfaces?
If a class implements multiple interfaces with conflicting default methods, it must override the method explicitly. The “class wins” rule applies: a method from a superclass takes precedence over interface defaults.
28. What is the purpose of the Collectors class in Java 8?
Collectors provides utility methods to collect Stream results into collections (e.g., toList(), toMap()), perform grouping (groupingBy()), or compute aggregates (summingInt()).
29. What are the advantages of using Optional over null checks?
Forces explicit handling of nullability, reduces NullPointerException risks, improves code readability with methods like orElse(), ifPresent().
30. What are the different ways to iterate over a collection using Java 8 features?
forEach(): list.forEach(System.out::println); Stream: list.stream().forEach(System.out::println); Iterator with lambda: iterator.forEachRemaining(System.out::println).
31. What is the difference between map() and flatMap() in Streams?
map(): Transforms each element into a single value (1:1 mapping). flatMap(): Transforms each element into a Stream and flattens it into a single Stream (1:many mapping). Example: flatMap() unwraps nested collections.
32. Explain the difference between findFirst() and findAny() in Streams.
findFirst(): Returns the first element in the Stream (deterministic). findAny(): Returns any element, useful in parallel Streams (non-deterministic).
33. How can you use reduce() in Java 8 Streams? 

        List numbers = Arrays.asList(1, 2, 3);
        int sum = numbers.stream().reduce(0, Integer::sum); // Output: 6
34. How does Java 8 improve garbage collection?
Java 8 replaced PermGen with Metaspace, which dynamically resizes and reduces memory errors. The G1 garbage collector also became the default, optimizing for low latency and large heaps.
35. Explain the difference between anyMatch(), allMatch(), and noneMatch() in Streams.
anyMatch(): Returns true if at least one element matches a predicate. allMatch(): Returns true if all elements match. noneMatch(): Returns true if no elements match. All return boolean.
36. How does Java 8 handle infinite Streams?
Infinite Streams are created with Stream.iterate() or Stream.generate(). They’re lazily evaluated, so operations like limit() or takeWhile() prevent infinite processing. Example: Stream.iterate(1, n -> n + 1).limit(5).
37. Explain the forEachOrdered() method in Streams.
forEachOrdered() ensures elements are processed in the encounter order, even in parallel Streams, unlike forEach(), which may process out of order in parallel.
38. How do you sort a list of objects using Streams in Java 8? 

        List people = ...;
        List sorted = people.stream()
            .sorted(Comparator.comparing(Person::getAge))
            .collect(Collectors.toList());
39. What is the peek() method in Streams used for?
peek() is an intermediate operation that allows inspecting or debugging elements without modifying the Stream (e.g., stream.peek(System.out::println)). It returns the same Stream.
40. What are the key differences between Java 7 and Java 8?
Java 8 introduces lambdas, Streams, and functional interfaces; default/static methods in interfaces; Optional for null handling; new Date/Time API (java.time); Metaspace instead of PermGen; enhanced collections and concurrency (e.g., CompletableFuture).

Collections & Data Structures (18 Questions)

1. What is the Java Collections Framework?
Classes and interfaces (e.g., List, Set, Map) for storing/manipulating objects.
2. What is the difference between ArrayList and LinkedList?
ArrayList: Dynamic array, fast access (O(1)), slow insertions (O(n)). LinkedList: Doubly-linked, slow access (O(n)), fast insertions (O(1)).
3. How does HashMap work internally in Java?
Array of buckets, keys hashed (hashCode()), collisions handled by linked lists (or trees in Java 8+).
4. What are the differences between HashMap, TreeMap, and LinkedHashMap?
HashMap: Unordered, O(1), allows nulls. TreeMap: Sorted, O(log n), no null keys. LinkedHashMap: Insertion order, O(1), allows nulls.
5. What is the difference between HashSet and TreeSet?
HashSet: Unordered, O(1), one null. TreeSet: Sorted, O(log n), no nulls.
6. How do you iterate over a collection in Java?
for loop, for-each, Iterator, ListIterator, or forEach() (Java 8+).
7. What are fail-fast and fail-safe iterators?
Fail-fast: Throws ConcurrentModificationException (e.g., ArrayList). Fail-safe: Works on copy or allows mods (e.g., CopyOnWriteArrayList).
8. How is ConcurrentHashMap different from HashMap?
Thread-safe, uses segment locking or CAS, no null keys/values.
9. What is a priority queue in Java?
Ordered queue (natural or custom order), implemented as a binary heap (PriorityQueue).
10. What is the difference between Queue and Deque?
Queue: Single-ended, FIFO. Deque: Double-ended, add/remove from both ends.
11. Explain Comparable and Comparator in Java.
Comparable: Natural ordering (compareTo()). Comparator: Custom ordering (compare()).
12. What is the difference between Stack and Queue?
Stack: LIFO. Queue: FIFO.
13. What are weak references in Java?
WeakReference allows GC to collect if no strong refs exist, used in caching.
14. Explain the internal implementation of ArrayList.
Dynamic array, starts at 10, grows 50% when full (new array copied).
15. What is the difference between a shallow copy and a deep copy?
Shallow: Copies references. Deep: Copies entire object graph.
16. What is the difference between Vector and ArrayList?
Vector: Synchronized, grows 100%. ArrayList: Not synchronized, grows 50%.
17. Explain the difference between IdentityHashMap and WeakHashMap.
IdentityHashMap: Uses == for keys. WeakHashMap: Weak keys, GC can remove entries.
18. What is EnumSet in Java?
Specialized Set for enums, efficient, uses bit vectors.

Multithreading (15 Questions)

1. What is a thread in Java?
Lightweight process for concurrent execution.
2. How do you create a thread in Java?
Extend Thread or implement Runnable, then start().
3. Explain the difference between Runnable and Thread.
Runnable: Interface with run(), reusable. Thread: Class implementing Runnable, creates thread.
4. What are daemon threads in Java?
Background threads (e.g., GC), terminate when user threads finish.
5. What is thread synchronization?
Coordinates access to shared resources using synchronized or locks.
6. What are the different types of thread synchronization?
Synchronized method, synchronized block, explicit locks (e.g., ReentrantLock).
7. What is a deadlock in Java?
Threads waiting indefinitely for each other’s resources.
8. What is a race condition, and how can it be avoided?
Unexpected behavior from unsynchronized access. Avoid with synchronization.
9. What are thread pools in Java?
Reusable threads managed by ExecutorService (e.g., newFixedThreadPool()).
10. What is the volatile keyword used for?
Ensures visibility across threads, prevents caching.
11. What is the difference between synchronized block and ReentrantLock?
Synchronized block: Implicit, simpler. ReentrantLock: Explicit, flexible (e.g., tryLock).
12. What is the ThreadLocal class in Java?
Thread-local variables, each thread has its own copy.
13. What is the ExecutorService framework?
Manages threads asynchronously (e.g., thread pools).
14. What is the difference between Callable and Runnable?
Callable: Returns value, throws exceptions. Runnable: Void, no exceptions.
15. How does the CountDownLatch work?
Threads wait until count reaches zero.

JVM & Garbage Collection (12 Questions)

1. What is the JVM, and how does it work?
Executes bytecode, provides platform independence via class loading, interpretation, and runtime management.
2. What are the different memory areas in JVM?
Heap (objects), Stack (method calls), Method Area (class data), PC Register, Native Method Stack.
3. What are garbage collection algorithms in Java?
Mark-and-Sweep, Copying, Mark-and-Compact.
4. What is the difference between Serial, Parallel, and G1GC collectors?
Serial: Single-threaded. Parallel: Multi-threaded, high throughput. G1GC: Region-based, low latency.
5. What is the difference between Stack and Heap memory?
Stack: LIFO, method-local, fast. Heap: Dynamic, object storage, GC-managed.
6. What is a memory leak in Java?
Unreferenced objects not reclaimed due to unintended references.
7. What is the difference between Strong, Weak, Soft, and Phantom references?
Strong: Prevents GC. Weak: GC collects if only weak refs. Soft: GC under pressure. Phantom: Notifies when GC’d.
8. How does the finalize() method work in Java?
Called by GC before reclaiming, unreliable, deprecated in Java 9.
9. What are Metaspace and PermGen?
PermGen: Pre-Java 8, fixed-size class metadata. Metaspace: Java 8+, dynamic, native memory.
10. What is the difference between minor GC and major GC?
Minor: Young generation. Major: Old generation (full GC).
11. How does the JVM optimize garbage collection?
Generational approach, JIT, adaptive algorithms.
12. What tools can be used for JVM monitoring?
JVisualVM, JConsole, JMX, VisualGC, GC logs.

Kafka & Messaging (10 Questions)

Distributed streaming platform for high-throughput messaging.

Producers send to topics (partitioned logs), consumed by consumers via brokers.

Topics split into partitions for parallelism, improves throughput.

SQL (15 Questions)

1. What is the difference between SQL and NoSQL databases?
SQL: Relational, structured, schema-based (e.g., MySQL). NoSQL: Non-relational, flexible (e.g., MongoDB).
2. What are primary keys and foreign keys in SQL?
Primary Key: Uniquely identifies rows, no nulls. Foreign Key: Links to primary key in another table.
3. What is the difference between INNER JOIN and LEFT JOIN?
INNER JOIN: Matching rows only. LEFT JOIN: All from left table, matched or NULL from right.
4. What are indexes in SQL, and why are they used?
Data structures for faster retrieval, improve query performance (e.g., B-tree).
5. What is normalization, and what are its forms?
Reduces redundancy: 1NF (atomic values), 2NF (no partial dependency), 3NF (no transitive dependency).
6. What is denormalization, and when is it used?
Adds redundancy for performance (e.g., reporting).
7. What is the difference between DELETE, DROP, and TRUNCATE?
DELETE: Removes rows, logs. DROP: Deletes table. TRUNCATE: Removes all rows, no logs, faster.
8. What are ACID properties in database transactions?
Atomicity, Consistency, Isolation, Durability.
9. What is a transaction in SQL?
Sequence of operations as a single unit (e.g., BEGIN, COMMIT, ROLLBACK).
10. What are SQL triggers?
Auto-executed procedures on events (e.g., INSERT, UPDATE).
11. What is the difference between UNION and UNION ALL?
UNION: Removes duplicates. UNION ALL: Keeps duplicates.
12. How do you optimize SQL queries?
Use indexes, avoid SELECT *, optimize joins, analyze EXPLAIN plans.
13. What is a subquery, and when should it be used?
Query within query, used for filtering or calculations (e.g., nested SELECT).
14. What is the difference between GROUP BY and ORDER BY?
GROUP BY: Aggregates data (e.g., SUM). ORDER BY: Sorts results.
15. What are stored procedures, and how do they differ from functions?
Stored Procedures: Precompiled SQL, no return. Functions: Return value, used in queries.

REST API (12 Questions)

1. What is REST, and what are its principles?
Representational State Transfer: Stateless, client-server, cacheable, layered, uniform interface (resources, HTTP methods).
2. What are the main HTTP methods used in REST APIs?
GET (retrieve), POST (create), PUT (update), DELETE (remove), PATCH (partial update).
3. What is the difference between PUT and PATCH?
PUT: Replaces entire resource. PATCH: Updates partial resource.
4. What are HTTP status codes, and what do they signify?
200 (OK), 201 (Created), 400 (Bad Request), 401 (Unauthorized), 404 (Not Found), 500 (Server Error).
5. What is statelessness in REST?
Each request contains all info needed, no server-side session.
6. How do you secure a REST API?
OAuth2, JWT, HTTPS, API keys, rate limiting.
7. What is HATEOAS in REST?
Hypermedia as the Engine of Application State: Responses include links for navigation.
8. What is the difference between REST and SOAP?
REST: Lightweight, resource-based, HTTP. SOAP: Protocol, XML, strict standards.
9. How do you handle versioning in REST APIs?
URI (e.g., /v1/), headers, or content negotiation.
10. What is content negotiation in REST?
Client specifies format (e.g., JSON, XML) via Accept header.
11. How do you implement pagination in a REST API?
Use query params (e.g., ?page=1&size=10) or offset/limit.
12. What are idempotent methods in REST?
Same result on repeated calls (e.g., GET, PUT, DELETE).

3. What is S3, and what are its use cases?
Simple Storage Service: Object storage for backups, static hosting.

Serverless compute, event-driven (e.g., file uploads).

7. What is Amazon RDS, and how does it differ from a traditional database?
Relational Database Service: Managed, auto-scaling vs. self-managed.
8. What is Elastic Load Balancer (ELB)?
Distributes traffic across EC2 instances for availability.
9. What is Auto Scaling in AWS?
Adjusts EC2 capacity based on demand.
10. What is VPC, and why is it used?
Virtual Private Cloud: Isolated network for security.
11. What are the differences between public cloud, private cloud, and hybrid cloud?

Design Patterns (18 Questions)

1. What are design patterns, and why are they important?
Reusable solutions to common problems; improve maintainability, scalability, and code readability.
2. What is the Singleton pattern, and how is it implemented in Java?
Ensures one instance; implemented with private constructor, static instance, and getInstance(). Example: 

public class Singleton {
    private static Singleton instance;
    private Singleton() {}
    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
3. What is the Factory Method pattern, and when should it be used?
Creates objects via subclasses; use when instantiation logic varies (e.g., creating different product types).
4. Explain the Prototype pattern with an example.
Clones objects; useful for costly creation. Example: 

public class Prototype implements Cloneable {
    @Override
    public Prototype clone() throws CloneNotSupportedException {
        return (Prototype) super.clone();
    }
}
5. What is the Builder pattern, and how does it help in object creation?
Constructs complex objects step-by-step; e.g., StringBuilder or creating immutable objects with many fields.
6. What is the Adapter pattern, and where is it used?
Converts incompatible interfaces; e.g., adapting legacy code to new systems.
7. What is the Decorator pattern, and how does it work?
Adds behavior dynamically; e.g., Java I/O streams (BufferedReader wraps Reader).
8. What is the Observer pattern, and how does it help in event handling?
Notifies dependents of state changes; e.g., java.util.Observer or GUI listeners.
9. Explain the Strategy pattern with an example.
Interchangeable algorithms; e.g., sorting: 

interface SortStrategy {
    void sort(int[] arr);
}
class BubbleSort implements SortStrategy {
    public void sort(int[] arr) { /* Bubble sort */ }
}
class QuickSort implements SortStrategy {
    public void sort(int[] arr) { /* Quick sort */ }
}
10. What is the Template Method pattern, and where is it used?
Defines algorithm skeleton; e.g., abstract class with hooks for subclasses.
11. What is the Facade pattern, and how does it simplify system design?
Unified interface to complex subsystems; e.g., JDBC API simplifies database access.
12. What is the Command pattern, and how does it help in request handling?
Encapsulates requests as objects; e.g., undo/redo in editors.
13. Explain the Chain of Responsibility pattern with an example.
Passes requests along handlers; e.g., logging levels: 

abstract class Logger {
    Logger next;
    void setNext(Logger next) { this.next = next; }
    abstract void log(String msg);
}
class InfoLogger extends Logger {
    void log(String msg) { if (msg.contains("INFO")) System.out.println(msg); else next.log(msg); }
}
14. What is the Proxy pattern, and how does it differ from Decorator?
Controls access (e.g., lazy loading); Decorator adds behavior.
15. What is the Flyweight pattern, and when should it be used?
Shares objects to save memory; e.g., String pool in Java.
16. What is the Composite pattern, and how is it used?
Treats individual and composite objects uniformly; e.g., UI component trees.
17. What is the Mediator pattern, and how does it reduce coupling?
Centralizes communication; e.g., chat room controller.
18. What is the State pattern, and how does it differ from Strategy?
Changes behavior based on state; Strategy changes algorithms independently.
